## Notes on style usage in this project

The codebase applies a consistent style influenced by the references above:

- attached / K&R brace style
- self-contained headers with include guards
- PascalCase for types and major functions
- snake_case for variables and data members
- trailing underscore for private class members

These rules were used to improve readability and consistency across the final codebase.
